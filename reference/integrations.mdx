<Warning>
  The projects listed on this page are community contributions. They are not maintained by the Headscale authors and are not verified or officially supported. They may be outdated or require additional steps.
</Warning>

## Reverse proxy

Running Headscale behind a reverse proxy lets you share a single external IP and port (typically `443`) with other services.

### Requirements

**WebSockets are required.** The reverse proxy must support WebSocket upgrades to communicate with Tailscale clients. WebSocket support is also needed for the embedded DERP server. You must also expose the STUN UDP port (default `udp/3478`) when using the embedded DERP server.

**Cloudflare is not supported.** Cloudflare proxies and tunnels do not support WebSocket POSTs as required by the Tailscale protocol.

### TLS configuration

When using a reverse proxy to handle TLS, configure Headscale to listen without TLS:

```yaml title="config.yaml"
server_url: https://<YOUR_SERVER_NAME>
listen_addr: 0.0.0.0:8080
metrics_listen_addr: 0.0.0.0:9090
tls_cert_path: ""
tls_key_path: ""
```

### Proxy configurations

<Tabs>
  <Tab title="nginx">
    Replace `<YOUR_SERVER_NAME>` with your FQDN, `<IP:PORT>` with the address Headscale is listening on (typically `localhost:8080`), and the certificate paths with your own:

    ```nginx title="nginx.conf"
    map $http_upgrade $connection_upgrade {
        default      upgrade;
        ''           close;
    }

    server {
        listen 80;
        listen [::]:80;

        listen 443      ssl http2;
        listen [::]:443 ssl http2;

        server_name <YOUR_SERVER_NAME>;

        ssl_certificate <PATH_TO_CERT>;
        ssl_certificate_key <PATH_CERT_KEY>;
        ssl_protocols TLSv1.2 TLSv1.3;

        location / {
            proxy_pass http://<IP:PORT>;
            proxy_http_version 1.1;
            proxy_set_header Upgrade $http_upgrade;
            proxy_set_header Connection $connection_upgrade;
            proxy_set_header Host $server_name;
            proxy_redirect http:// https://;
            proxy_buffering off;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
            add_header Strict-Transport-Security "max-age=15552000; includeSubDomains" always;
        }
    }
    ```
  </Tab>
  <Tab title="Caddy">
    Caddy automatically provisions TLS certificates and proxies WebSockets. Replace `<YOUR_SERVER_NAME>` with your FQDN and `<IP:PORT>` with the address Headscale is listening on (typically `localhost:8080`):

    ```text title="Caddyfile"
    <YOUR_SERVER_NAME> {
        reverse_proxy <IP:PORT>
    }
    ```

    No further configuration is required — Caddy handles HTTPS and WebSocket proxying automatically.
  </Tab>
  <Tab title="Apache">
    Note the `upgrade=any` parameter on `ProxyPass`. This is required so that WebSocket traffic whose `Upgrade` header is not equal to `WebSocket` (i.e. Tailscale Control Protocol) is forwarded correctly.

    ```apache title="apache.conf"
    <VirtualHost *:443>
        ServerName <YOUR_SERVER_NAME>

        ProxyPreserveHost On
        ProxyPass / http://<IP:PORT>/ upgrade=any

        SSLEngine On
        SSLCertificateFile <PATH_TO_CERT>
        SSLCertificateKeyFile <PATH_CERT_KEY>
    </VirtualHost>
    ```
  </Tab>
  <Tab title="Istio / Envoy">
    Istio and Envoy require an additional upgrade type to avoid `upgrade_failed` errors.

    **Envoy**: Add a new `upgrade_type` named `tailscale-control-protocol` to your `HttpConnectionManager` config. See the [Envoy docs](https://www.envoyproxy.io/docs/envoy/latest/api-v3/extensions/filters/network/http_connection_manager/v3/http_connection_manager.proto#extensions-filters-network-http-connection-manager-v3-httpconnectionmanager-upgradeconfig) for details.

    **Istio**: Use an `EnvoyFilter` to add the upgrade type:

    ```yaml
    apiVersion: networking.istio.io/v1alpha3
    kind: EnvoyFilter
    metadata:
      name: headscale-behind-istio-ingress
      namespace: istio-system
    spec:
      configPatches:
        - applyTo: NETWORK_FILTER
          match:
            listener:
              filterChain:
                filter:
                  name: envoy.filters.network.http_connection_manager
          patch:
            operation: MERGE
            value:
              typed_config:
                "@type": type.googleapis.com/envoy.extensions.filters.network.http_connection_manager.v3.HttpConnectionManager
                upgrade_configs:
                  - upgrade_type: tailscale-control-protocol
    ```
  </Tab>
</Tabs>
